Overview

  • View your family inheritance with cylc graph -n:

cylc validate -v -v

  • Plain cylc validate command:
    • tells you if your suite will be able to run
    • ...& if not, why (at least first issue pinpointed)
  • Then turn the verbosity up to 11!
    • warnings, information, file paths etc. printed
    • see e.g. deprecated settings, 'unsafe' or experimental usage instances & expansion processing of namespaces & Jinja2

cylc graph -n

cylc graph -n

  • DO:
    • refer to (& obey) the Suite Design Guide
    • use root namespace appropriately i.e. to set global defaults for settings used by the vast majority of tasks
    • create a sensible structure: maximise sharing whilst minimising overriding & namespace coupling & surfeit
    • give families helpful names e.g. HPC
  • DON'T:
    • ...do the opposite of the above (e.g. overuse root & create defaults not used by most tasks)
